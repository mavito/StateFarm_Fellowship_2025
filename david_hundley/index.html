System Analysis & Design in AI/ML at State Farm

About the Speaker

David Hundley is a Staff Machine Learning Engineer at State Farm, where he has worked since 2012 in various roles including Scrum Master and Business Architect.

He currently leads the technical Generative AI (GenAI) strategy for the company, focusing on model selection, appropriate use cases, and creating best practice guides for implementation.

Outside of State Farm, David maintains a blog and YouTube channel teaching AI/ML concepts.


A High Level Intro to AI/ML

David provided a comprehensive overview of the AI landscape

  • Artificial Intelligence (AI): A broad field of science aiming to create intelligent machines that simulate cognitive abilities.
  • Machine Learning (ML): A subset of AI focusing on mathematical algorithms that learn from data and improve through experience without explicit programming .
  • Deep Learning (DL): A specialized form of ML using multi-layered artificial neural networks, excelling at processing complex patterns in large datasets.
  • Generative AI (GenAI): A type of AI that can create new content, including text, images, audio, and video, based on patterns learned from existing data.

Example of a popular GenAI image generation tool.

Case Study: ECC AI Responses

Problem Statement

State Farm’s Enterprise Care Center (ECC) assists agents and team members with technical problems via phone or IM. Representatives rely on a robust knowledge base to answer questions, but finding the right article can be challenging, especially for new employees.

The Solution: Retrieval Augmented Generation (RAG)

To solve this, the team implemented a RAG paradigm, which retrieves appropriate context from the knowledge base to generate intelligent responses.

The Core Retrieval Augmented Generation (RAG) Paradigm.

Process Flow

The system integrates into the agent's workflow. The AI checks if a question is business-related before producing a response, which the ECC representative then verifies for validity before sending it to the agent.

ECC AI Responses Process Map.

System Architecture

The technical implementation utilizes Amazon Web Services (AWS), including Lambda, API Gateway, OpenSearch Vector DB, and LLMs via AWS Bedrock.

High-level System Architecture.

Project Execution & Team Composition

The project followed Agile delivery practices, including 2-week sprints, regular stand-ups, backlog grooming, and retrospectives. The team composition included:

  • Project Manager / Product Owner: Leads effort, communicates with stakeholders.
  • Scrum Master: Facilitates agile ceremonies and removes roadblocks.
  • UX Designers: Defines look-and-feel and user interaction.
  • Software Engineers: Builds and supports the application.

Building Competency

David outlined key competencies required for success in AI/ML system design:

  • Project Management: Agile practices, Gantt charts, and effective communication.
  • Product Ownership: Story writing, backlog grooming, and scope protection.
  • Technical: Understanding data (structured vs. unstructured), AI risks/capabilities, agile software (Jira, GitLab), and pursuing certifications like AWS Certified Cloud Practitioner.
